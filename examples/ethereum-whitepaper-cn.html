01 / INTRODUCTION

介绍

以太坊的开发背景与哲学

📜 比特币的历史

中本聪在2009年开发的比特币，是首个将去中心化货币概念变为现实的项目。比特币的一个重要创新是区块链技术，它允许网络就交易状态达成共识。

💡
区块链的核心价值

区块链作为一个分布式账本，解决了拜占庭将军问题，实现了在无需信任第三方的情况下进行价值转移。

🎯 以太坊的开发哲学

以太坊的设计哲学是创建一个通用的区块链平台，而不是专门为某种特定应用设计。通过提供图灵完备的编程语言，开发者可以构建任意复杂的智能合约和去中心化应用。

02 / BITCOIN CONCEPTS

比特币与现有概念

理解区块链技术的核心概念

🔄 状态转换系统

比特币可以被视为一个状态转换系统：给定一个状态 S 和一个交易 TX，系统会产生一个新的状态 S'。

状态转换函数 Pseudocode 
// 比特币状态转换
APPLY(S, TX) -> S' = {
    if !isValidTX(TX, S):
        return ERROR
    return newState(S, TX)
}

⛏️ 挖矿机制

比特币使用工作量证明（PoW）机制来达成共识。矿工通过解决数学难题来竞争记账权，成功者获得比特币奖励。

参数 值 说明
区块时间 ~10分钟 平均出块时间
区块奖励 6.25 BTC 每区块奖励（2024）
最大供应 2100万 比特币总量上限

🌳 默克尔树

默克尔树是一种数据结构，用于高效地验证大量数据的内容。在比特币中，所有交易都被组织成默克尔树，树根存储在区块头中。

⚠️
SPV 验证

简化支付验证（SPV）允许轻量级客户端验证交易，而无需下载完整的区块链数据。

👤 以太坊账户

以太坊有两种类型的账户：外部拥有账户（EOA）由私钥控制，合约账户由代码控制。每个账户都有余额、nonce和可选的代码/存储。

📨 消息和交易

交易是外部账户发送的签名数据包，包含：接收者地址、发送者签名、价值（ETH）、数据、gas限制和gas价格。

交易结构 Solidity-like 
struct Transaction {
    address to;         // 接收者
    uint256 value;      // 转账金额
    bytes data;         // 输入数据
    uint256 gasLimit;   // Gas 上限
    uint256 gasPrice;   // Gas 价格
    uint256 nonce;      // 交易序号
    bytes signature;    // 签名
}

⚡ 代码执行 (EVM)

以太坊虚拟机（EVM）执行智能合约字节码。EVM是图灵完备的，支持循环和递归，但受gas限制保护免受无限循环攻击。

💡
Gas 机制

Gas 是以太坊中计算资源的计量单位。每个操作都有固定的gas成本，防止滥用并确保网络可持续运行。

05 / CONCLUSION

结论

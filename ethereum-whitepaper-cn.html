下一代智能合约和去中心化应用平台

1. 介绍

中本聪在2009年开发的比特币，是首个将去中心化货币概念变为现实的项目。比特币不仅创造了一种新的货币形式，更重要的是，它展示了区块链技术的巨大潜力。

然而，比特币的设计主要用于货币交易，其脚本语言功能有限。以太坊的目标是建立一个通用的区块链平台，支持任意复杂的计算，使开发者能够构建各种去中心化应用。

💡 核心创新
以太坊引入了图灵完备的编程语言，允许开发者编写智能合约，实现任意复杂的逻辑，而不受比特币脚本语言的限制。

2. 比特币与现有概念

2.1 区块链技术历史

区块链技术是去中心化共识的产物。它通过密码学方法和分布式网络，实现了在没有中心化信任机构的情况下，达成全网一致的状态。

核心概念包括：

  • 🔗分布式账本：所有交易记录存储在网络中的每个节点
  • 🔐密码学哈希：确保数据完整性和不可篡改性
  • 🤝共识机制：网络节点就账本状态达成一致

2.2 作为状态转换系统的比特币

从技术角度看，比特币可以理解为一个状态转换系统。系统状态由所有已存在的未花费交易输出(UTXO)组成。

// 状态转换函数 APPLY(S, TX) → S' // 其中： // S = 当前状态 // TX = 交易 // S' = 新状态 function applyTransaction(state, tx) { // 1. 验证输入签名 for (input in tx.inputs) { verifySignature(input); } // 2. 检查输入是否未被花费 for (input in tx.inputs) { assert(state.unspent[input]); } // 3. 更新状态 return newState(state, tx); }

2.3 挖矿

比特币使用工作量证明(PoW)机制来达成共识。矿工通过解决数学难题来创建新区块，并获得比特币奖励。

挖矿参数 说明 当前值
区块时间 生成新区块的平均时间 ~10分钟
区块奖励 矿工获得的比特币数量 6.25 BTC
难度调整 保持稳定出块时间的机制 每2016块
最大供应量 比特币总量上限 2100万

2.4 默克尔树

默克尔树(Merkle Tree)是一种二叉树数据结构，用于高效验证大量数据的完整性。

🌳 默克尔树优势
  • 高效验证：只需 O(log n) 个哈希即可验证某个交易
  • 节省空间：轻节点只需存储区块头
  • 防篡改：任何数据修改都会导致根哈希变化

2.5 替代区块链应用

比特币的成功激发了众多替代区块链应用的出现：

  • 📛Namecoin - 去中心化域名系统
  • 🎨Colored Coins - 在比特币上发行自定义资产
  • 📊Metacoins - 在比特币之上构建新协议

2.6 脚本

比特币的脚本语言存在以下限制：

  • ❌ 缺乏图灵完备性（无循环语句）
  • ❌ 盲目性（无法访问区块链数据）
  • ❌ 价值盲性（无法查询UTXO金额）
  • ❌ 缺乏状态（无法保存内部状态）

3. 以太坊

3.1 以太坊账户

以太坊使用账户模型，而非比特币的UTXO模型。每个账户包含：

字段 类型 说明
nonce uint64 交易计数器，防止重放攻击
balance uint256 账户余额（以Wei为单位）
code bytes 智能合约字节码
storage mapping 合约存储的键值对
🔐 账户类型
外部账户(EOA)：由私钥控制，无代码
合约账户(CA)：由代码控制，有存储

3.2 消息和交易

以太坊中的交易包含以下字段：

struct Transaction { address recipient; // 接收方地址 uint256 value; // 转账金额（Wei） bytes data; // 附加数据 uint256 gasLimit; // Gas限制 uint256 gasPrice; // Gas价格 uint64 nonce; // 交易序号 bytes signature; // 数字签名 }

3.3 以太坊状态转换函数

以太坊的状态转换函数执行智能合约代码：

function applyTransaction(state, tx) { // 1. 检查nonce和签名 assert(tx.nonce == state[sender].nonce); verifySignature(tx); // 2. 扣除Gas费用 gasFee = tx.gasLimit * tx.gasPrice; state[sender].balance -= gasFee; // 3. 执行转账 state[sender].balance -= tx.value; state[recipient].balance += tx.value; // 4. 执行合约代码（如果有） if (state[recipient].code.length > 0) { executeCode(state[recipient].code, tx.data); } // 5. 返回剩余Gas state[sender].balance += remainingGas * tx.gasPrice; return state; }

3.4 代码执行

以太坊使用以太坊虚拟机(EVM)执行字节码。EVM是一个图灵完备的虚拟机，具有以下特点：

  • ⚡基于栈的架构
  • 💾临时内存和持久化存储
  • ⛽Gas机制防止无限循环
  • 🔒沙盒环境隔离执行

3.5 区块链和挖矿

以太坊最初使用工作量证明，后转向权益证明。采用了基于GHOST协议的共识机制，提高了网络安全性。
